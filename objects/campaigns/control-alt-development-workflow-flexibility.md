## Purpose

Make the Control Alt development workflow less dependent on any single editor, coding agent, provider, connector, or private search mechanism.

## Problem

A healthy development workflow should survive tool churn.

When workflow continuity depends too heavily on one editor, one coding agent, one connector, or one private indexing path, the work becomes brittle. The goal is not to abandon all existing tools immediately. The goal is to make Praxis the durable workflow authority so multiple humans and agents can participate safely.

## Campaign intent

This Campaign evaluates and improves:

- agent flexibility,
- provider cost awareness,
- editor neutrality,
- public/private context boundaries,
- evidence preservation,
- repeatable review and validation,
- safety boundaries for private implementation work.

## Public Praxis role

Public Praxis is part of the Campaign because replacement agents need stable public orientation before they are tested against private implementation context.

A useful agent should be able to load this public repo and understand:

- what Praxis is,
- what Campaigns, Beacons, Strands, Fibers, Directives, Readouts, Probes, Protocols, Doctrines, and Imprints are,
- what belongs in public vs private Praxis,
- why public Praxis is not operational authority,
- why private implementation work requires separate bounded context.

## Success criteria

This Campaign is moving in the right direction when:

- Praxis remains the durable workflow authority,
- public Praxis gives agents a stable read-only map,
- private Praxis remains private and operational,
- Directives preserve instruction semantics,
- Readouts capture real agent/provider/workflow evaluation results,
- Relay or any transport layer remains transport, not authority,
- implementation work can be evaluated without depending on a single tool.

## Non-goals

This public Campaign summary does not authorize:

- private repo access,
- implementation changes,
- publishing private operational details,
- merge/deploy/runtime/device/network/Docker/destructive actions,
- treating public Praxis as the private ledger of record.
