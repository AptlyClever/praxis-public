# Praxis Public

Praxis is the Control Alt workflow ledger that turns intent into auditable execution.

This is **Praxis Public** — the public-readable orientation layer.
It contains durable workflow canon, object model definitions, current work summaries,
and agent orientation prompts.

Praxis Public is not the operational ledger.
Private Praxis retains authority for handoffs, implementation, deployment, and runtime decisions.

## Lifecycle

```
Signal → Response → Campaign → Aim → Strand → Stint → Grant → Execution → Resolution
```

## Object kinds

| Kind | Meaning |
| --- | --- |
| **Signal** | Low-authority captured idea or observation preserved for later triage |
| **Campaign** | Coordinated multi-lane effort spanning agents, repos, and decision checkpoints |
| **Beacon** | Current navigation, state, and evidence map for a Campaign or Strand |
| **Strand** | A continuing line of attention kept open toward an outcome; may be active, paused, or completed |
| **Fiber** | Small identifiable work unit that advances a parent Strand |
| **Directive** | Instruction object: authorized action, explicit boundaries, validation expectations, stop conditions |
| **Readout** | Recorded result from a bounded evaluation run of an agent, provider, workflow, or related system |
| **Probe** | Reusable check — observation-first pass/fail evidence |
| **Protocol** | Repeatable procedure composed of steps an operator or agent can run |
| **Doctrine** | Reusable guidance that outlives a single task |
| **Imprint** | Durable rationale or decision trace |

## Object storage pattern

```
objects/<kind>/<human-title-slug>.md
```

## Authority boundary

Public Praxis is for orientation and context only.

- Does not grant authority to modify private repos.
- Does not authorize deployment, service operation, or runtime changes.
- Does not replace private Praxis as the operational ledger.
- Implementation requires a private Directive from private Praxis.
